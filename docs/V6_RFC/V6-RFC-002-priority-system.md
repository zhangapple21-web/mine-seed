# V6-RFC-002: 任务优先级体系（A > C > B）

**状态**: Draft  
**作者**: 疯子(CCO)  
**日期**: 2026-06-16  
**依赖**: RFC-001

---

## 1. 优先级定义

| 类别 | 优先级 | 含义 | 调度规则 |
|------|--------|------|----------|
| **A-生产** | P0 | 必须执行，不能丢 | 到点立即执行，可抢占低优先级 |
| **C-维护** | P1 | 低频重要 | A类来了让路，空闲时执行 |
| **B-探索** | P2 | 有更好，没也行 | 永远给A/C让路，可随时被抢占 |

## 2. 任务分类清单

### A类：生产任务

| 任务 | Cron | 时间窗口 | 说明 |
|------|------|----------|------|
| 矿场v5 | 每4h | 00/04/08/12/16/20 | 核心收入引擎 |
| Stock Advisor | 每日 | 08:15 | 必须在09:30前推出 |

## 3. 调度规则

### 3.1 抢占规则

```
A类任务到达 → 立即检查云电脑状态
  ├─ 空闲 → 直接执行
  ├─ 被A类占用 → 排队等待（同优先级FIFO）
  ├─ 被C类占用 → C类暂停，A类执行，C类重新排队
  └─ 被B类占用 → B类终止，A类执行，B类丢弃（B可重跑）
```

### 3.2 时间窗口约束

Stock Advisor必须在09:30前完成，这是硬约束。如果08:15触发时云电脑被其他A类占用，等待但不超过08:45。

### 3.3 Cron避让

```
时间轴：
  04:00  矿场v5 + 信号发现（可并行，不抢云电脑）
  08:15  Stock Advisor（独占云电脑）
  09:40  Dragon Leader早盘
  12:00  矿场v5
  13:30  Dragon Leader午盘
  14:50  Dragon Leader尾盘
  16:00  矿场v5 + 信号发现
  20:00  矿场v5
  20:04  档案官
  22:00  信号发现
  00:00  矿场v5
```

B类任务只能在A/C类的时间窗口间隙执行。

## 4. 冲突解决

| 冲突场景 | 解决方案 |
|----------|----------|
| 两个A类同时触发 | 先到先执行，后到排队 |
| A类到达时B类正在跑 | B类终止，不保存中间结果 |
| C类到达时B类正在跑 | B类等C类完成后再继续 |
| 心跳检查 vs 其他 | 心跳不占云电脑，无冲突 |

## 5. 约束生成

当Scheduler检测到：
- 活跃Session > 50 → 自动禁止新的B类spawn
- 活跃Session > 30 → 自动禁止新的C类spawn
- A类任务延迟 > 5min → 自动终止所有B/C类
