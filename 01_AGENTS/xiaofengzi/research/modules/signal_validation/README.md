# M01 信号验证模块

**模块ID**: M01
**职责**: 验证某个信号是否真有效、有效区间在哪里、失效边界是什么
**核心产出**: 验证报告 + 置信度评级
**状态**: v0.1 迭代中

---

## 验证SOP（标准流程）

### Step 1: 问题定义
- 信号名称/描述
- 预期作用（选股/择时/风险控制）
- 背后的逻辑假设
- 参考来源

### Step 2: 数据准备
- 数据区间
- 样本范围（全市场/某板块/某市值区间）
- 数据来源说明

### Step 3: 验证方法
- 统计方法（相关性/回测/事件研究等）
- 对照组设置
- 评价指标（胜率/盈亏比/夏普/最大回撤等）

### Step 4: 分层验证
1. **全样本验证**：整体是否有效？
2. **时间分层**：不同年份/牛熊周期表现是否一致？
3. **市值分层**：不同市值区间表现差异？
4. **行业分层**：不同行业表现差异？
5. **极端值检验**：极端行情下是否失效？

### Step 5: 失效边界探索
- 什么情况下这个信号会失效？
- 失效的预警信号是什么？
- 失效是暂时的还是永久性的？

### Step 6: 结论输出
- 是否有效：是/否/部分有效
- 有效区间：什么市场环境/什么类型标的
- 置信度：高/中/低（基于证据强度）
- 风险提示：主要局限性和可能的偏差

---

## 置信度评级标准

| 等级 | 标准 |
|------|------|
| 高 | 多时间段验证一致 + 跨市场有效 + 逻辑链条完整 + 排除过拟合 |
| 中 | 基本验证通过，但某些区间表现不稳定，或逻辑链条有假设成分 |
| 低 | 初步观察有相关性，但未经过严格验证，或样本量不足 |
