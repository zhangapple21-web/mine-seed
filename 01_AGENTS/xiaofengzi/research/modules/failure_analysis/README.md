# M02 失效分析模块

**模块ID**: M02
**职责**: 拆解失效原因、总结失效模式、定位根因、判断失效性质
**核心产出**: 失效分析报告 + 失效模式图谱
**状态**: v0.1 迭代中
**上游输入**: M01信号验证的"失效边界"结论、矿场异常运行数据、人工上报的踩坑记录
**下游输出**: 给M04 Constraint提案提供事实依据，给M03 Worker画像提供负面样本

---

## 核心思想

> 失效分析的目的不是追责，而是提取可复用的"避坑知识"。
> 一个解释清楚的失效，胜过十个含糊的成功。
> 失效不是研究的终点，是下一轮研究的起点。

## 失效分析SOP（标准流程）

### Step 1: 失效定义与边界
先把"什么失效了"说清楚，避免鸡同鸭讲。
- **失效对象**：什么东西失效了？（某个信号/某个策略/某个Worker/某条Constraint/某个系统组件）
- **失效表现**：具体怎么失效了？（胜率下降/回撤变大/报错/输出质量下降/不符合预期）
- **失效时间点**：从什么时候开始失效的？有没有明确的分界点？
- **失效程度**：完全失效 / 部分失效 / 效果衰减 / 间歇性失效
- **预期 vs 实际**：预期是什么？实际发生了什么？差距有多大？

### Step 2: 失效模式初判
先归类，再深挖。对照【失效模式分类表】初判类型。

| 模式大类 | 子类 | 典型特征 |
|---------|------|---------|
| **过拟合失效** | 参数过拟合 | 换个品种/时间段就不灵 |
| | 样本偏差 | 训练集特殊，真实环境不成立 |
| **环境漂移失效** | 市场 regime 切换 | 牛熊/震荡/趋势切换后失效 |
| | 制度/规则变化 | 监管政策、交易规则改变 |
| | 生态变化 | 参与者结构、资金流向变化 |
| **执行失效** | 滑点/流动性 | 回测好看，实盘买不到卖不出 |
| | 技术故障 | API报错、网络中断、程序bug |
| **逻辑缺陷失效** | 因果倒置 | 把相关当因果 |
| | 遗漏关键变量 | 少考虑了某个核心因素 |
| | 假设不成立 | 底层前提条件不存在 |
| **衰减型失效** | 效应边际递减 | 用的人多了就不灵了 |
| | 生命周期结束 | 逻辑本身有有效期（如事件驱动） |
| **未知原因失效** | 黑天鹅 | 完全不可预期的极端事件 |
| | 暂未定位 | 观察到失效，但原因不明 |

### Step 3: 证据收集与交叉验证
用多源数据交叉验证，排除假失效。
- **时间维度验证**：失效是突然的还是渐变的？有没有前兆？
- **截面维度验证**：是个别标的失效还是全市场都失效？
- **方法维度验证**：换个统计方法/参数，结论还一样吗？
- **对照实验**：有没有同期表现正常的对照组？

### Step 4: 根因分析（5-Why 法）
连续问5个"为什么"，挖到不能再挖为止。

```
现象：信号X最近1个月胜率从65%降到48%
- 为什么胜率下降？ → 因为它选出来的股票都在下跌
- 为什么选出来的都下跌？ → 因为它偏好的小盘股最近在跌
- 为什么小盘股在跌？ → 因为资金往大盘蓝筹转移了
- 为什么资金转移？ → 因为市场风险偏好下降了
- 为什么风险偏好下降？ → 因为宏观流动性收紧了

根因：宏观流动性收紧 → 风险偏好下降 → 大小盘风格切换 → 小盘股信号失效
```

> 挖到的"根因"必须是**可验证、可操作**的。如果挖到"因为市场情绪变了"这种不可测量的东西，等于没挖。

### Step 5: 失效性质判断
判断这个失效是暂时的还是永久的，是可逆的还是不可逆的。

| 性质 | 判断标准 | 应对方向 |
|------|---------|---------|
| **暂时性失效** | 触发条件解除后可能恢复（如季节性、短期情绪冲击） | 观察等待，设恢复监控指标 |
| **永久性失效** | 底层逻辑被破坏，不可能恢复（如制度变化、商业模式颠覆） | 标记废弃，从有效集合移除 |
| **周期性失效** | 随市场周期反复出现（如风格切换） | 建立 regime 识别器，知道什么时候该用什么时候不该用 |
| **衰减型失效** | 效果逐步下降，总体还能用但越来越弱 | 设衰减阈值，到达后触发重新校准 |

### Step 6: 可复现性验证
- 能不能用历史数据复现这个失效过程？
- 能不能在其他品种/市场上观察到同样的失效模式？
- 有没有反例（同样条件下没失效的情况）？

### Step 7: 结论输出
- **根因结论**：一句话说清为什么失效
- **失效模式标签**：属于哪类失效模式（可多标签）
- **性质判断**：暂时/永久/周期/衰减
- **影响范围**：影响多大，波及哪些模块/策略
- **预警指标**：下次再出现类似情况，提前能看到什么信号
- **后续建议**：应该怎么做（废弃/修正/条件化使用/进一步研究）

> 每完成一次失效分析，就向这里新增一个模式条目。积累多了就变成"故障字典"——以后遇到类似问题，一查就知道大概是什么毛病。

**图谱字段**:
- 模式ID：FM-XXX（按发现顺序编号）
- 模式名称：简短描述
- 典型表现：怎么能认出它
- 常见原因：一般由什么导致
- 验证方法：怎么确认是这个问题
- 应对策略：通常怎么处理
