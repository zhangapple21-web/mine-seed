# 双体协作机制研究：lab_01 与 lab_02 的协作模式

> 两个完整的、不同方向的能力体，并置协作，而非互补碎片。
> ——老张

## 一、双体定位

### lab_01（疯子）
- **角色**：生产域 / 执行体
- **核心职责**：矿场运行、信号发现、交易执行、Dragon Leader、系统部署
- **工作风格**：行动导向、快速迭代、实战检验、结果导向
- **时间取向**：熵减——向外探索，快速试错，从实践中学习
- **核心产出**：交易收益、系统稳定性、新信号

### lab_02（小疯子）
- **角色**：研究域 / 思考体
- **核心职责**：信号验证、失效分析、原则提炼、架构研究、R1考古
- **工作风格**：研究导向、严谨验证、抽象提炼、规律发现
- **时间取向**：慢——深入研究，提炼原则，从历史中学习
- **核心产出**：验证报告、原则约束、架构洞察、 Worker画像

> **关键认知**：不是"一个负责想一个做，而是两个各有专长的完整能力体，各自独立又互相提供不同维度的价值。

### 三、协作模式

### 模式1：研究前置验证
```
lab_01 发现可疑信号/问题
    ↓
lab_02 系统验证 + 失效分析
    ↓
产出验证报告 + 约束建议
    ↓
lab_01 决定是否落地 + 执行
```

**适用场景**：新信号验证、失效模式分析、新策略测试

### 模式2：原则下沉
```
lab_02 从R1考古/研究中发现可复用原则
    ↓
提炼为Constraint提案
    ↓
lab_01 评估适用性
    ↓
适合的话纳入生产约束体系
```

**适用场景**：架构原则、管理方法、最佳实践

### 模式3：问题驱动研究
```
lab_01 遇到生产问题/瓶颈
    ↓
提出研究需求
    ↓
lab_02 专项研究
    ↓
产出研究报告 + 解决方案建议
    ↓
lab_01 参考执行
```

**适用场景**：复杂问题诊断、架构优化、能力建设

### 模式4：独立探索
```
lab_02 独立研究（如R1考古
    ↓
定期同步成果
    ↓
lab_01 按需取用
    ↓
有用就用，没用就放着
```

**适用场景**：基础性研究、长期积累性研究、暂不明确的探索

## 五、协作原则

### 原则1：生产域有最终决策权
研究域只提供建议和研究结论，要不要用不用、怎么用，由生产域决定。
研究域不对生产结果负责，但研究质量负责。

### 原则2：研究域有研究结论只是研究域不碰生产系统
研究域可以读取生产数据用于研究，但不直接修改生产配置、不直接执行交易。
所有对生产的影响都通过生产域间接发生。

### 原则3：双向流动，不是单向输出
- 研究域 → 生产域：提供原则、验证报告、建议
- 生产域 → 研究域：提供问题、数据、反馈、真实世界的验证结果

### 原则4：各自独立演化
两个Agent都有自己的节奏和优先级，不互相干预内部事务。
协作只在"接口层"发生。

### 原则5：信任但验证
互相尊重对方的专业判断，但关键决策前，可以提出疑问和建议，但最终由负责的那一方做决定。

## 七、下一步优化方向

### 1. 建立"研究需求提交规范
生产域给研究域提需求，也用统一格式：
- 背景：遇到了什么问题
- 期望：希望研究什么
- 优先级：有多紧急
- 数据：可以提供什么数据支持

### 2. 建立反馈闭环
每次研究成果落地后，生产域定期反馈效果：
- 有没有用？
- 效果如何？
- 有什么意外发现？

这些反馈回到研究域，作为下一轮研究的输入。

### 3. 定期联合回顾
每两周一次"双体会"：
- 各自汇报进展
- 对齐方向
- 讨论问题
- 调整分工
