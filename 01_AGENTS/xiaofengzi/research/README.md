# 研究域模块化架构 v0.1

**状态**: 实验版 · 迭代中
**原则**: 单一职责 · 独立迭代 · 一致性兜底

---

## 架构三层

```
┌─────────────────────────────────────────────────┐
│  路由层 (Router)                                │
│  • 接收研究任务                                 │
│  • 判断研究类型 → 分发给对应模块               │
│  • 最后过一致性审查                             │
└─────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────┐
│  模块层 (Modules) — 单一职责，独立迭代          │
│  ┌──────────┬──────────┬──────────┐           │
│  │ 信号验证  │ 失效分析  │ Worker画像│           │
│  └──────────┴──────────┴──────────┘           │
│  ┌──────────┬──────────┬──────────┐           │
│  │Constraint│ 同构检测  │ R1考古   │           │
│  │  提案    │          │          │           │
│  └──────────┴──────────┴──────────┘           │
└─────────────────────────────────────────────────┘
                        │
                        ▼
┌─────────────────────────────────────────────────┐
│  证据层 (Evidence) — RAG素材库                   │
│  • 历史研究报告 • 矿场运行数据                   │
│  • 外部参考资料 • Principle/Constraint全集      │
└─────────────────────────────────────────────────┘
```

## 模块清单与优先级

| 优先级 | 模块ID | 模块名称 | 核心职责 | 产出物 | 状态 |
|--------|--------|----------|----------|--------|------|
| **P0** | M02 | 失效分析模块 | 拆解失效原因、失效模式、规律总结 | 失效分析报告 + 失效模式图谱 | ✅ v0.1 SOP已搭 |
| **P1** | M04 | Constraint提案模块 | 从证据提炼可执行约束，分A/B类 | Constraint草案 + 生命周期管理 | ✅ v0.1 SOP已搭 |

| **P4** | M05 | 同构检测模块 | 发现重复研究，合并同构结论 | 去重合并建议 | ⏳ 待搭建 |

> 发现错误 → 解释错误 → 形成约束 → 改变未来行为
> 这才是真正的学习。

---

## 工作流

1. **输入**：新发现 / 新数据 / 新问题
2. **路由**：判断属于哪类研究 → 分配对应模块
3. **执行**：模块按自身SOP产出结果
4. **审查**：过一致性审查，检查与已有Principle是否冲突
5. **归档**：结果存入证据层，作为后续研究的素材
