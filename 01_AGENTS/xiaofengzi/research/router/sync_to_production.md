# 研究域 → 生产域 同步机制

> 研究的终点不是报告，而是生产行为的改变。
> 研究域产出 → 生产域落地 → 结果反馈 → 再研究，形成闭环。

---

## 一、同步内容分类

### 第一类：Constraint / 规则类
- **内容**：验证通过的约束、原则、边界规则
- **优先级**：高（直接影响生产决策）
- **同步方式**：主动推送 + 版本化
- **接收方**：生产域Governance模块

### 第二类：Worker画像 / 能力边界
- **内容**：各Worker的适用场景、成功率、失效模式
- **优先级**：中（影响任务分配效率）
- **同步方式**：定期更新 + 异常触发
- **接收方**：生产域Router模块

### 第三类：信号验证结果
- **内容**：新信号的验证结论、已有信号的有效性追踪
- **优先级**：高（直接影响交易策略）
- **同步方式**：事件驱动 + 完整报告
- **接收方**：生产域Signal模块

### 第四类：架构/方法论参考
- **内容**：R1考古发现、架构原则、最佳实践
- **优先级**：低（长期参考，不直接影响当前生产）
- **同步方式**：定期汇总 + 按需查阅
- **接收方**：生产域全局

## 二、同步触发条件

### 主动同步（研究域发起）
1. **Constraint成熟时**：当一条Constraint从冷却期进入观察期/生效期时，主动推送给生产域
2. **重大发现时**：信号验证有重大结论（如某信号完全失效/效果超预期）
3. **Worker画像重大更新时**：某Worker能力边界有重大变化
4. **定期汇总**：每周一汇总上周的研究成果，统一推送

### 被动同步（生产域请求）
1. **生产域遇到问题时**：主动来研究域找答案
2. **需要验证某个假设时**：生产域提出验证需求
3. **架构调整时**：需要参考研究结论做决策

## 三、同步格式规范

所有同步内容统一格式，方便生产域处理：

```
【研究域同步】类型：[Constraint/信号/画像/参考]
标题：[简短描述]
版本：v1.0
日期：YYYY-MM-DD
置信度：[高/中/低]
摘要：[1-2句话说清核心内容]
详细内容：
  - [要点1]
  - [要点2]
  - [要点3]
落地建议：
  - 建议怎么用
  - 注意事项
  - 预期影响
附件：[完整报告链接/路径]
```

## 四、反馈闭环

同步不是单向推送，必须有反馈闭环：

1. **生产域收到后**：给一个确认回执（"已收到，将在X个工作日内评估"）
2. **评估后**：反馈是否采纳、计划怎么落地
3. **落地后**：反馈实际效果如何
4. **效果数据回流**：回到研究域，用于进一步验证和迭代

> 没有反馈的同步 = 没有同步。
> 研究域不能自嗨，必须知道自己的研究有没有用、有多大用。

## 七、注意事项

1. **不打扰生产**：研究域同步不能影响生产域正常运行。非紧急内容走定期汇总渠道。
2. **置信度透明**：每个结论都要标明置信度，不能把猜测当定论。
3. **版本化管理**：每次更新都有版本号，生产域可以选择用哪个版本。
4. **有进有出**：不仅研究域往生产域推，生产域也要往研究域提需求、反馈结果。
5. **不越界**：研究域只提供研究结论和建议，不直接修改生产域配置。
