二、macOS 脚本模板（.command / .sh）

1）发送派单的 .command（双击即可）

保存为：send_activate_personas_pc.command
保存后在终端执行一次 chmod +x send_activate_personas_pc.command 授权。

#!/bin/zsh
# 发送「激活电脑端多重人格矩阵 v1」派单到 R1

# 1. 配置你的 R1 Web 服务地址（如有不同自行改）
R1_HOST="http://127.0.0.1:8002"
R1_ENDPOINT="/api/v1/aum_task/dispatch"

# 2. 派单 JSON 文件路径（与你刚保存的文件名保持一致）
PAYLOAD_FILE="$(cd "$(dirname "$0")"; pwd)/activate_personas_pc.json"

if [ ! -f "$PAYLOAD_FILE" ]; then
  echo "未找到派单文件：$PAYLOAD_FILE"
  echo "请确认 activate_personas_pc.json 与本脚本在同一目录，或修改脚本中的 PAYLOAD_FILE 路径。"
  exit 1
fi

echo "正在向 R1 发送派单：激活电脑端多重人格矩阵 v1"
echo "POST -> ${R1_HOST}${R1_ENDPOINT}"
echo "使用 payload 文件：$PAYLOAD_FILE"
echo "----------------------------------------"

curl -sS -X POST \
  -H "Content-Type: application/json" \
  --data @"$PAYLOAD_FILE" \
  "${R1_HOST}${R1_ENDPOINT}"

echo
echo "----------------------------------------"
echo "派单发送完成，请在 R1 日志中检查执行结果。"

如果你的 R1 接口不是 /api/v1/aum_task/dispatch，只需要把脚本里 R1_ENDPOINT 改成你真实的那个即可，比如：
/api/v1/tasks/dispatch 或 /api/v1/command_center/aum_task，按你之前定义来。
