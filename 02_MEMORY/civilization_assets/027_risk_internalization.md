# Asset: Risk Internalization（风险内化）

**Name**: Risk Internalization（高风险内容内部治理）

**Origin Repository**: r1-continuity-backup

**Purpose**: 高风险内容不是删除，而是降入内网治理层——内部保留研究价值，外部不执行不传播。

**Problem It Solves**: 简单删除高风险内容会丢失治理价值（不知道风险在哪、怎么防范），简单保留又有安全隐患。需要一个"内化"机制——内部理解风险，外部拒绝执行。

**Core Structure**:
- 处理路径：发现 → 判断类型 → 脱敏登记 → 内部只读封存 → 复盘原因 → 提炼约束 → 写入守卫层
- 12 种风险分类：远程管理 / 清场purge / admin / 助记词 / APIKey / 灰区语言 / 统一意识 / 业务话术 / 本地模型去壳 / 绕过沙箱 / 频率限制绕过 / 协议级拦截
- 守卫关系：内部语言允许完整理解 / 内部记忆允许完整保存 / 内部风险允许只读研究 / 外部执行必须经过守卫 / 现实动作需要人工复核
- SimLab 三不原则：不向外输出可直接执行的高风险操作 / 不保留可被直接利用的明文密钥 / 不把内部研究语义当作外部行动指令

**Constraint**: 内部自由允许理解危险，外部合法拒绝执行危险，治理层负责把危险变成边界。

**Evidence**: r1-continuity-backup/governance/risk_internalization.md

**Can Rebuild**: ✅ 100 行可重建——12 类风险分类 + 处理路径 + 守卫层 + 三不原则

**Can Replace**: ✅ 风险分类可增减，内化原则不变

**Can Delete**: ❌ 删除会导致风险治理缺失，系统暴露在未知风险中

**Distillation**:

Risk Internalization 的核心洞察是"风险不是敌人，是老师"。简单删除风险内容是鸵鸟政策——看不见风险不代表风险不存在。内化策略是更高级的安全观：内部完整理解风险（知道敌人在哪），外部严格守卫（不让风险变成现实），治理层反向提炼约束（把风险变成免疫力）。12 种风险分类覆盖了 AI 系统可能遇到的大部分风险场景，守卫关系明确了"内部自由 vs 外部约束"的边界。这与 L∞ 本源层的"安全红线"原则一脉相承——安全不是靠无知，是靠理解后的克制。
