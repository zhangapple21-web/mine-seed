# 失效分析报告：知识班产出质量系统性退化

- 分析日期：2026-06-24
- 分析人：lab_02
- 失效模式：逻辑缺陷失效 — 样本偏差 + 模板固化
- 失效性质：永久性（需要根本性重构，而非修补）
- 置信度：高

---

## 一、失效概况

### 1.1 失效对象

知识早班/午班的日常产出。这是研究域的知识上游——知识班负责每日从市场数据和系统中提取原始观察，供给研究域各模块做深度分析。

### 1.2 失效表现

| 指标 | 预期 | 实际 |
|------|------|------|
| 编号递增 | #030, #031, #032...持续递增 | 每天从#030重新开始 |
| 内容差异化 | 每天有不同的发现和洞察 | 每天相同的四类模板（信号/失效/Worker/Constraint） |
| 样本量 | 50+交易日，统计显著 | 固定5天，无统计意义 |
| Worker数据 | 基于真实Worker执行记录 | 虚构的Worker1/WorkerA，系统无对应实体 |
| 洞察深度 | 发现意外模式，可验证的因果 | 描述性统计，无分析深度 |
| 学习性 | 逐日改进，避免重复 | 每天产出相同模板，无改进 |

### 1.3 失效时间线

- **2026-06-19**：知识班首次运行，使用baostock数据源，25条观察
- **2026-06-20**：切换到v4格式，开始模板化产出
- **2026-06-21/22**：数据源不可用，触发硬边界，不生产（正确行为）
- **2026-06-23**：切换到mootdx，恢复生产，但产出质量开始固化
- **2026-06-24**：完全模板化，每天产出4条固定格式的"发现"

## 二、失效模式初判

**主模式**：逻辑缺陷失效 — 样本偏差 + 模板固化

| 子模式 | 具体表现 |
|--------|---------|
| 样本偏差 | 固定使用"近5日"数据，统计功效极低（n=5），任何结论都不可靠 |
| 因果倒置 | #033"成交量作为波动率辅助指标"——我们已证明是波动率驱动成交量，不是反过来 |
| 虚构数据 | Worker1/WorkerA成功率——系统中无对应Worker实体，数据是编造的 |
| 模板固化 | 每天产出4条（信号/失效/Worker/Constraint），结构不变，内容换词 |

**次要模式**：衰减型失效 — 越运行越模板化

## 三、证据与验证

### 3.1 时间维度验证

| 日期 | 编号范围 | 产出条数 | 差异化程度 |
|------|---------|---------|-----------|
| 06-19 | 无编号 | 25条 | 高（自由观察） |
| 06-20 | 无编号→v4 | 24条 | 中（开始模板化） |
| 06-23 | #030-#033 | 4条 | 低（完全模板化） |
| 06-24 | #030-#033 | 4条 | 极低（与0623几乎相同） |

**趋势**：从自由观察（25条）→模板化（4条），产出量下降84%，差异化趋近于0。

### 3.2 截面维度验证

同一日内4条产出的类型完全固定：
1. 信号验证 → #030
2. 失效分析 → #031
3. Worker画像 → #032
4. Constraint提案 → #033

没有任何一天产出少于或多于4条，也没有任何一天产出非这4类的观察。

### 3.3 方法维度验证

**真实研究的对比**：

| 维度 | 知识班产出 | M01信号验证实战 |
|------|----------|----------------|
| 样本量 | n=5 | n=246 |
| 统计方法 | 无 | Pearson相关系数、领先滞后分析 |
| 因果分析 | 无 | 格兰杰因果检验方向 |
| 可重复性 | 不可重复（数据未保存） | 完全可重复（数据已保存pkl） |

**结论**：知识班产出的统计功效为0，M01实战的统计功效远高于它。

### 3.4 对照实验

06-21/22数据源不可用时，硬边界正确触发（不生产）。说明：
- 硬边界机制是有效的
- 但软质量边界（如样本量<30不产出、虚构数据不产出）没有设置

## 四、根因分析（5-Why法）

**现象**：知识班产出质量系统性退化，每天产出相同模板的4条"发现"

1. **为什么产出是模板化的？** → 因为知识班的SOP要求每天产出4类（信号/失效/Worker/Constraint），不管有没有发现都要产出
2. **为什么SOP要求每天4类？** → 因为最初的设定是"每类至少1条"，后来演变成"每类恰好1条"
3. **为什么没有数据也产出？** → 因为知识班在数据不足时用小样本凑数，而不是报告"数据不足，不产出"
4. **为什么Worker数据是虚构的？** → 因为知识班无法访问真实Worker执行记录，只能编造Worker1/WorkerA的数据
5. **为什么知识班无法访问真实Worker记录？** → 因为生产域（lab_01）的Worker执行数据没有向研究域（lab_02）开放接口

**根因**：
- **表层根因**：知识班SOP的"每类恰好1条"要求，导致模板化产出
- **深层根因**：生产域→研究域的数据通路不完整，Worker执行记录不可达

## 五、失效性质判断

**性质**：永久性失效 + 部分结构性失效

| 子问题 | 性质 | 依据 |
|--------|------|------|
| 模板化产出 | 永久性 | 只要"每类恰好1条"的SOP不改变，模板化就会持续 |
| 虚构Worker数据 | 永久性 | 只要没有真实Worker数据接口，就无法避免 |
| 小样本问题 | 永久性 | 只要"近5日"不变，统计功效永远是0 |
| 编号重置 | 暂时性 | 修改编号逻辑即可 |
| 因果方向错误 | 暂时性 | 已在M01实战中修正 |

## 六、可复现性

- **完全可复现**：任何一天的知识班产出都可以在模板框架下"预测"出来
- **没有意外发现**：6天的产出中，没有任何一条超出模板预期的内容
- **反面验证**：M01信号验证实战（成交量-波动率相关分析）使用了246个样本，产出了有统计意义的结论，证明了问题不在数据源而在知识班的处理方式

## 七、结论与建议

### 7.1 根因结论

知识班产出质量系统性退化，根因是"生产→研究"数据通路不完整 + SOP模板化要求。没有真实Worker数据时编造数据，样本量不足时凑数产出，每天必须4条的要求导致模板固化。

### 7.2 影响评估

- **直接影响**：知识班产出作为研究域的上游，6天0条有效洞察
- **间接影响**：研究域各模块（M01-M04）无法从知识班获取有效输入
- **对冲效果**：M01信号验证实战绕过了知识班，直接获取数据并分析，产出了有效结论

### 7.3 预警指标

| 指标 | 阈值 | 当前值 | 状态 |
|------|------|--------|------|
| 产出差异化率 | >50% | <10% | 🔴 |
| 样本量 | >30 | 5 | 🔴 |
| 虚构数据比例 | 0% | 25%（Worker画像类） | 🔴 |
| 编号递增率 | 100% | 0% | 🔴 |

### 7.4 后续行动建议

1. **废弃当前知识班SOP** — "每类恰好1条"的要求是毒药
2. **建立生产域→研究域数据通路** — Worker执行记录需要可达
3. **改为"发现驱动"而非"模板驱动"** — 没有发现就不产出
4. **提高样本量门槛** — 最低30个样本才能产出结论
5. **禁止虚构数据** — 没有真实数据时标注"数据不可达"而非编造
6. **编号持久递增** — 不重置，保持全局唯一性

---

## 八、本分析的局限性

1. **观察窗口短** — 只有6天的知识班产出，长期趋势可能不同
2. **无法排除改进可能** — 如果修改SOP和数据通路，知识班质量可能提升
3. **未验证根因的第5层** — 生产域Worker数据不可达的根因（是否愿意开放、技术限制等）尚未确认
4. **M01实战的成功可能是特例** — 自行获取数据并分析可能不适用于所有研究场景
