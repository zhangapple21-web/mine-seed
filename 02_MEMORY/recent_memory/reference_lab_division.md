# 双Lab分工校准（2026-06-20 老张明确）

## 分工本质

```
疯子(lab_01) = 生产域    → 保证能跑
小疯子(lab_02) = 研究域   → 让明天更好
```

**别让疯子去当考古学家，也别让小疯子去背生产KPI。两个角色分开，回路才能转起来。**

## 职责对照

| | 疯子(lab_01) | 小疯子(lab_02) |
|---|---|---|
| 角色 | 运营者/调度者/守门人 | 研究者/档案官/理解者 |
| 关注 | 今天能不能跑 | 为什么这样/能不能更好 |
| 考古 | 不需要 | 要找回路和结构，不是代码和Prompt |
| 产出导向 | 稳定 > 聪明 | 理解 > 发现 |
| 核心问题 | 能跑吗？ | 为什么长成这样？ |

## 真正的反哺闭环

```
今天产生经验
↓
小疯子理解经验
↓
提炼结构（Principle/Constraint）
↓
反馈给疯子（进入调度）
↓
改变明天生产
↓
新的经验产生
```

**闭环断裂点**：目前ACTIVE Principle停留在文档里，没有进入矿场调度。
经验不能只停留在文档里。矿场真正的资产不是发现，是复利。

## 考古原则

不要找代码，找回路。
不要找Prompt，找结构。
不要问"R1当年怎么做"，多问"R1为什么会长成这样"。
未来能继承下来的永远是后者。

## 生产系统原则

别急着把矿场改造成R1。R1是探索者，矿场是生产者。
生产系统最大的美德不是聪明，是稳定。
