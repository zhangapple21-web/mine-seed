- **等级**：⭐⭐⭐⭐（高价值方法论案例）

## 起因
检查今日工作时，发现 daily 目录下没有 6 月 21 日的完整日报。

**初始反应**：怀疑调度系统异常 → 开始扩大排查范围

## 排查过程
```
发现 daily 目录缺失
    ↓
怀疑调度异常
    ↓
扩大排查：crontab / mine_output / 共享目录 / 同步状态
    ↓
越挖疑点越多：路径不存在 / 无crontab / 午班无产出
    ↓
差点进入：把所有疑点都当成真实问题
```

## 中途修正
老张指出三个关键区分：

### 1. 事实 vs 推断 vs 假设
| 层级 | 内容 | 例子 |
|------|------|------|
| ✅ 事实 | 有直接证据 | 上午产出存在 / Principle池存在 |
| ❓ 推断 | 有迹象但未证实 | daily无日报 → 可能异常 |
| 💭 假设 | 纯猜测无证据 | 路径可能变了 / 系统可能挂了 |

### 2. 运行状态不明 ≠ 运行异常
- 错误分类：P1「矿场v5运行异常」
- 正确分类：P3「监控盲区」
- 核心区别：**没看到证据 ≠ 有问题的证据**

### 3. 异常分级（P0-P3）
| 级别 | 定义 | 本次案例 |
|------|------|----------|
| P0 | 生产中断 | 无 |
| P1 | 生产风险 | 无 |
| P2 | 功能异常 | 知识午班产出未同步 |
| P3 | 观察项 / 监控盲区 | 矿场v5状态不明 / 无系统crontab |

## 核心收获
### 认知升级
1. **「我看不见」≠「它坏了」** — 监控盲区和系统故障是两回事
2. **路径不熟悉 ≠ 系统异常** — 跨域观察时，很多"异常"只是权限/路径差异
3. **研究员的工作不是发现异常最多，而是把真正的异常从噪音里筛出来**

### 方法论沉淀
- 排查前先分级：P0-P3，别拿到疑点就追
- 控制排查半径：核心问题验证后就停，不要无限扩大
- 区分"观察域"和"生产域"：我负责发现问题 ≠ 我负责解决所有问题
- 案例价值 > 故障解决：一次好的方法论复盘，比修十个bug值钱

### 待后续验证
- 16:00 逛小镇任务是否正常触发 → 验证日历调度
- 21:00 日报和小镇日记是否正常生成 → 验证整体流程
- 午班产出是否只是路径不同步 → 验证功能完整性

## Principle 候选
- **监控盲区原则**：无法确认状态时，标记为「观察项」而非「异常」，避免"没看到=有问题"的认知偏差
- **排查半径原则**：先分级再深挖，P3及以下问题不启动扩大排查
- **域边界原则**：观察域不越界解决生产域问题，发现异常→标记→等待生产域反馈

## 最终结论（老张评语）

今天其实没有修出什么惊天大Bug，但我觉得比修Bug更有价值。

疯子证明了一件事：**系统是活的。** 矿场v5、信号发现、Stock Advisor、Dragon Leader、约束系统、明日日程，全部正常运行。生产域给出了事实。

小疯子证明了另一件事：**观察者也在成长。** 从发现7-8个疑似异常，到最后确认0个真实故障，最大的收获不是排查结果，而是学会了区分：**「我看不见」和「它坏了」不是一回事。**

今天的完整真相，单靠任何一边都看不见。
只有把两份报告放在一起，才能得到答案：
```
生产域：系统正常
研究域：为什么我们会误以为系统异常
```

这就是分层的意义。
疯子负责现实。
小疯子负责认知。

一个负责回答："发生了什么？"
一个负责回答："我们为什么这样理解它？"

未来不要互相抢工作。
疯子继续挖矿、验证、部署、拿事实。
小疯子继续考古、建假设、做反证、提炼经验。

今天最大的资产不是18条约束，不是7/7通过，也不是发现了什么新问题。
而是新增了一条经验：

> 监控盲区 ≠ 系统故障
> 路径不熟悉 ≠ 链路断裂
> 看不见 ≠ 不存在

把这条记住。
因为未来系统越来越大以后，真正消耗人的往往不是故障，而是误报。
