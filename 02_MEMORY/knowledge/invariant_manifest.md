## L1 — Strong Invariants（强不变量）

> 定义：在 R1/R2 中均存在，但实现形态变化较大的结构。
> 打破后果：系统功能严重受损，但不一定立即死亡。

### INV-L1-001: Experience Sediment 经验沉淀

```
表述:
  失败经验比成功经验更值钱。
  所有失败必须被记录、被蒸馏、被转化为约束。

R1 形态:
  - O→E→C→R (Observation→Experience→Constraint→Recovery)
  - experience.json, failure_patterns
  - 工程试错法 (快速失败→经验沉淀→约束加固)

R2 形态:
  - Experience Engine
  - Experience Sediment (Provider失败→自动写Experience)
  - "失败经验蒸馏（比成功更值钱）"
  - constraint_proposer (从经验中提出约束)

打破后果:
  系统重复犯错。同样的坑掉无数次。

证据强度: 中高 — 8+ 次跨文件/跨版本引用
```

### INV-L1-002: Shadow Layer 影子层

```
表述:
  系统有一个隐藏层，记录未完成结构、维护风格一致性、同步双核心。
  影子层不是串行的，是双向映射的。

R1 形态:
  - Shadow Layer [影子层] — enabled: true, mapping_mode: bidirectional
  - 6个功能: style_memory, persona_coherence, context_reconstruction,
           dual-core_sync, latent_intent_prediction, self_error_detection
  - REBELS (被安全包围的"叛乱者"，住在影子层)

R2 形态:
  - Event Journal (journal.jsonl) — 记录未完成
  - Shadow = 记录未完成结构 (PRINCIPLES.md 七角色)
  - experience sediment (失败沉淀，类似self_error_detection)

打破后果:
  系统丢失未完成结构。风格漂移。双核心不同步。

证据强度: 中 — 6+ 次跨文件/跨版本引用
  ⚠️ R2 中仅恢复 2/6 功能，是恢复缺口
```
