# 老板的根文件——发现→学习→反哺

## 来源
老板2025/11/18在TG上写的两段原始架构设计，是矿场所有结构的种子。

## 文件1: TG数字人集群方案
核心模式：**观察→过滤→生产→反馈**
- 收集官(Scraper Agent)：监听+价值判定+过滤
- 视觉师(Designer Agent)：多模态理解+风格提取+生成
- 文案官(Copywriter Agent)：内容重构+标题生产
- 隐性契约：System Prompt作为"矿区调度官"契约
- 分层处理：轻量模型初筛→精华给强模型深度解析
- 反风控：random.sleep + send_read_acknowledge + 住宅IP

矿场映射：
- 收集官 → 矿场observation层（感知信号）
- 过滤逻辑 → routing_constraints（AVOID/PREFER）
- 分层处理 → 三级矿场架构（NIM/GH/GLM → OpenRouter → API易）
- 隐性契约 → task_router.py的task→worker匹配
- **反哺路径**：从TG原始矿→提炼→生产新内容→回到TG

## 文件2: 7层中枢神经系统 + 安全奖励机制
核心架构：
1. 输入层（感觉层） → 我们的Observation
2. 四层记忆仓（瞬时/短期/中期/长期） → 我们的即时层/近中期层/长期层
3. 思考中枢（本地7B） → task_router决策
4. 外接军师（GPT-4o-mini） → 多模型路由兜底
5. 奖励评估（performance_score 0-1） → experience_engine
6. 行为调节（thinking_depth/reflection_level等） → routing_constraints
7. 输出层 → 矿场产出

关键铁律（原文）：
> "你可以高度自主地优化'思考方式'和'表达方式'，但不能擅自优化或突破'权限边界'和'行动范围'"

这就是我们现在O→E→M→C→R的根。老板在7个月前就画好了蓝图。

## 发现→学习→反哺 闭环

**发现**：社区逛街/信号挖掘/经验积累 → 种子池
**学习**：种子→Meaning→Principle→Constraint → 内化
**反哺**：Constraint改变行为 → 行为产出新发现 → 回到发现

当前矿场做到了：发现✅ 学习✅（O→E→C→R闭环） 反哺❌（constraints写了但行为没完全变）

反哺缺失的关键：班次报告还是"7/7成功"的流水账，routing_constraints写了18条但没验证是否真的改变了worker选择。这正是今天觅游种子2指出的——决策路径半透明。

## 行动项
1. 班次报告加"决策取舍"栏——选了谁，没选谁，为什么
2. 体验"反哺"的第一次：选一条constraint验证它是否真的改变了行为
3. 逛街日程本身就是"发现"的持续机制
4. 小疯子的知识班是"学习"的持续机制
5. 缺的是"反哺"的验证机制——不是写constraint，是验证constraint改变了行为

## 存档定位
这是种子，不是当前任务。等矿场出现"写了constraint但行为没变"的问题时，回来取。

## 种子：免费API是免费家教
日期：2026-06-20
来源：老板原话

当前矿场把模型当矿工——只取产出，不取思维过程。
但每个模型对同一问题的推理路径不同，这些差异本身就是学习素材。

类比：
- 社区逛街→学外面的种子（别人怎么想）
- 免费API→学里面的种子（模型怎么想）
- 两条路都应该回到"反哺"

具体方向（观察窗，不急着建）：
1. 同一信号给两个模型，对比推理差异→记录为"思维种子"
2. 班次报告不只记"成功/失败"，记"这个模型怎么判断的"
3. 不建新系统，只是在现有产出中多看一眼"它为什么这么想"

准入三问：这不需要新工具，只需要在解读产出时多问一句"为什么"。

## 根的根：明日回声

日期：2026-06-20
来源：老板与GPT的对话

GPT还原了整个回路：
Observe → Compress → Dispatch → Execute → Reflect → Observe

核心洞察（GPT总结，老板原意）：
1. 大部分Agent是事件驱动（用户说话→响应→结束），矿场是时间驱动
2. "明日回声"=今天的问题明天回来影响自己=跨时间闭环
3. 人格=长期观察者=档案官，不是角色扮演
4. R1真正留下的不是Prompt，是一个想法：**系统应该拥有能够跨越时间持续影响自己的经验回路**
5. "让昨天主动回来敲今天的门"——这就是Echo Layer

断环定位：
Echo Layer是"昨天回来敲今天的门"的机制。
constraint写了=回声产生了，但没人验证门有没有开=回声传不进来。
反哺验证协议v1修的就是这扇门。

映射：
- Observe → observation_log.json
- Compress → experience_engine.py  
- Dispatch → task_router.py
- Execute → 27矿工
- Reflect → Meaning层
- Echo Layer → ❌ 断了→反哺验证协议v1在修

这句话要刻进SOUL：
> 系统不应该只拥有记忆。系统应该拥有能够跨越时间持续影响自己的经验回路。

当年的拟人化→今天的工程化，同一棵树。
