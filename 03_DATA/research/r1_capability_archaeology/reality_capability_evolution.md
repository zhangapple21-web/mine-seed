
## 七、最终结论

### 7.1 回答三个关键问题

**问题一：ACE 当年为什么会大量接浏览器？**

**答案：浏览器是最大的数据源和最大的 Runtime。**

- 网页包含几乎所有公开信息
- 浏览器可以执行 JavaScript，操作 DOM
- 浏览器是最通用的用户界面
- 证据：`memory_index_latest.json` 中的 Selenium 采集记录

**问题二：为什么后来不用了？**

**答案：稳定性、权限、维护成本、账号登录、反爬等问题。**

- 直接证据：`2026-06-21-morning-heartbeat.md` 记录浏览器登录需要手机号验证码
- API DNS 解析失败
- 反爬机制越来越严格
- 维护成本高

**问题三：今天重新看，哪些应该回来？**

**答案：经过能力蒸馏后的抽象层应该回来。**

| 技术 | 是否应该回来 | 原因 |
|------|------------|------|
| Playwright | ✅ | 跨浏览器、稳定、统一 API |
| Browser MCP | ⚠️ 待验证 | 标准化但需要评估安全性 |
| WebMCP | ❌ 暂不 | 还不成熟 |
| Selenium | ❌ 淘汰 | 维护成本高 |
| IE COM | ❌ 淘汰 | 技术已过时 |

### 7.2 核心结论

**R2 真正要保存的不是 Browser，不是 MCP，不是任何框架，而是能力（Capability）。**

技术会不断更替，但能力会沉淀为文明资产。只要 Capability DAG 建立起来，未来无论出现什么新技术，它们都只是对同一组能力的不同实现。
